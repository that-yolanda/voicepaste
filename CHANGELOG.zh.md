# 更新说明

## v2.1.1 (2026-06-29)

快捷键可靠性修复版本。

### Fix

- **快捷键失效** — 修复开启 CapsLock / NumLock 等按键时 toggle 快捷键静默失效的问题（这类按键的抬键信号常被系统吞掉，会卡住状态机）。现在在源头忽略非快捷键按键，并在空闲几秒后自动回收卡住的按键。

## v2.1.0 (2026-06-27)

本次发布聚焦性能与稳定性：录音主路径全面脱离 WebView（音频采集与提示音移至 Rust 后端），macOS 悬浮窗不再依赖 WebView、设置页改为按需加载，空闲内存下降约 80MB；同时新增转写失败重试、后端快捷键录制等能力。

### New Features

- **原生音频采集** — 麦克风采集全部迁移至后端 cpal（macOS CoreAudio / Windows WASAPI），取代 WebView 的 getUserMedia。录音音量不再「前轻后响」，解决开头识别不准；并省去浏览器音频收敛延迟，启动更跟手。
- **提示音优化** — 开始/结束提示音改由后端 rodio 播放，与麦克风采集通道分离。原生采集无回声消除，开始提示音会漏入麦克风，因此主动跳过开头片段，避免提示音被误识别为语音；同时播放更稳定准时，不受 WebView 销毁影响。
- **快捷键录制迁移后端** — 快捷键录制改由后端 keytap 完成，可绑定 DOM 录制无法捕获的硬件级按键（如 macOS Fn 键）。
- **内存优化** — 设置页 WebView 改为懒加载（关闭即销毁而非隐藏），macOS 悬浮窗移除 WebView 依赖，空闲状态内存减少约 80MB。
- **可选保留识别音频** — 「设置 → 应用设置」新增「保留录音」选项，识别音频可在首页历史记录查看/播放；识别失败时自动保留音频以支持重试。
- **转写失败重试** — 在线 ASR 连接失败 / 超时 / 无结果时，可再次按快捷键或点击重试，用留存的音频重新转写，无需重录；commit 超时改为返回错误而非回退残缺文本，避免不完整结果被当成成功粘贴。
- **语音输入弹框界面优化** — 优化录音弹框的提示状态展示，窗口缩放增加过渡动画，提示文字增加呼吸效果，使输入反馈更流畅。

### Fix

- **音频波形** — 波形动画更流畅（各条按独立频段 + 非对称包络），并统一 macOS / Windows 的波形高度计算逻辑。
- **设置页置顶** — 修复启动时与中途打开设置窗口未置顶的问题。
- **快捷键符号显示** — 修复 Windows 下按键符号误显为 macOS 键盘符号的问题，改为按平台显示原生符号。
- **快捷键冲突与竞态** — 重构为 chord 状态机：toggle 按 keyup 周期触发、hold 按 keydown 触发，以录音期间按下的最长组合决定最终输出；解决前缀冲突（如 Ctrl 抢占 Ctrl+Shift）与快速连按的时序问题。

## v2.0.0 (2026-06-17)

> **⚠️ 重大变更**：VoicePaste 已从 Electron 完全重写为 Tauri v2（Rust 后端），这是一次重大架构变更。首次启动时会自动迁移 1.x 配置。

### 架构升级

- **Electron → Tauri v2** — 整个应用使用 Tauri v2（Rust + WebView）重建，安装包从 ~120 MB 缩减至约 20 MB，空闲内存从 ~500 MB 降至约 80 MB。
- **Apple 官方签名与公证** — macOS 安装包经过 Apple Developer 签名及公证，安装时不会触发 Gatekeeper 安全警告。
- **React / TypeScript 前端重写** — 设置页面使用 React + TypeScript + Tailwind CSS 重建，统一设计系统，支持深色/浅色/跟随系统主题。

### ASR 语音识别

- **在线 / 本地双引擎** — 在线引擎：火山引擎豆包流式语音识别（支持新的 API Key 鉴权方式）；本地引擎：基于 sherpa-onnx，支持 4 种离线模型。
- **4 种本地模型** — SenseVoice（中英日韩粤）、Zipformer 中英双语、FunASR-Nano（中英 + 7 种方言）、Qwen3-ASR-0.6B（30 种语言）。本地模型按需加载，内存 500 MB 起，支持 CPU / CUDA / CoreML 加速。
- **VAD + 标点还原** — 内置 Silero VAD（语音活动检测）进行音频分段；可选配 Punctuation 模型做识别后处理，追加标点符号。
- **模拟流式输出** — 对不支持原生流式输出的本地模型，通过 VAD 分段实现模拟流式输出，识别结果实时呈现。
- **智能重连** — 在线模型断连后自动重连，保障长时间录音稳定性。
- **新老版本鉴权兼容** — 同时支持火山引擎新的 API Key 方式和老的 APP ID / Access Token / Secret Key 方式。

### LLM 文本润色

- **8 家 LLM 提供商** — DeepSeek、OpenAI、Anthropic、Gemini、OpenRouter、SiliconFlow、Ollama 及 OpenAI 兼容接口。
- **自定义提示词模板** — 内置通用整理、翻译、邮件撰写等场景模板，支持自定义 Prompt，可为不同模板绑定独立快捷键。
- **热词强化** — 支持三种模式（自动 / 开启 / 关闭），可将热词追加到 LLM 提示词中以强化识别准确率。

### 热词库

- **多组热词表** — 支持创建多个热词组，可手动切换启用。
- **权重参数** — 支持 `热词|权重值` 格式，权重范围 1–10。
- **批量添加** — 通过逗号分隔一次性添加多个热词。
- **热词替换** — 识别完成后自动还原热词的原始格式（大小写、标点符号等），无需手动修正。
- **在线热词表** — 火山引擎在线热词表与本地热词表可同时生效，本地优先级更高。

### 快捷键系统

- **keytap 原生引擎** — 替换 uiohook-napi / tauri-plugin-global-shortcut，支持纯修饰键触发、左右修饰键精确区分。
- **Toggle / Hold 双模式** — 点击切换（按一次开始、再按一次结束）和按住说话（按住开始、松开结束），中途可按 ESC 取消。
- **场景独立快捷键** — 可为不同文本润色模板绑定独立快捷键，每个模板支持独立的触发模式。

### 配置系统

- **registry.json 模型注册表** — 所有模型的默认参数统一管理，用户只需在 config.yaml 中覆盖需要修改的字段。
- **三层合并** — 共享默认 → 模型专属默认 → 用户覆盖，未设置的字段自动回退到默认值。
- **即时生效（Hot Reload）** — 保存配置后无需重启应用，修改立即生效。
- **1.x 配置迁移** — 首次启动 v2.0 时自动导入 Electron 版本的配置文件。

### 其他

- **Beta 更新通道** — 设置中开启后可接收 Beta 版本更新提醒，尝鲜新功能。Stable 用户不受影响。
- **结构化日志** — 分级模块前缀日志，支持文件轮转（300KB，gzip 压缩），便于排查问题。
- **提示音自定义** — 支持替换默认的开始/结束提示音为本地的音频文件。
- **保留剪切板** — 启用后识别结果保留在剪切板中，不自动粘贴，可手动粘贴到任意位置。
- **macOS Liquid Glass 浮窗** — 原生 AppKit Liquid Glass 透明浮窗，支持浅色/深色/系统主题。
- **配置文档** — 新增 [GUIDANCE.zh.md](GUIDANCE.zh.md) 用户配置指南，覆盖快速开始、模型配置、热词、LLM 等全部功能。
